### B2.3-009: Spreadtrum as separate Vendor enum value

- **Status:** Confirmed (source is correct, docs are inconsistent)
- **Severity:** Medium
- **Location:** lib/include/mbootcore/domain/DeviceTypes.hpp:19 vs docs
- **Authority:** DeviceTypes.hpp — Spreadtrum = 6 is a separate enum value from UNISOC = 3
- **Evidence:** Source code defines both `UNISOC = 3` and `Spreadtrum = 6` as separate enum values. DeviceDiscovery.md correctly lists both. However, Overview.md:234 says "UNISOC (Spreadtrum)" implying they're the same. HardwareTesting.md:66 says "UNISOC / Spreadtrum".
- **Recommendation:** Keep both enum values as they are in source. Update Overview.md and HardwareTesting.md to clarify that UNISOC and Spreadtrum are separate vendor entries (UNISOC is the current company name, Spreadtrum is the legacy name — both exist in the enum for compatibility).
- **Target B3 Patch:** Patch 2

### B2.3-010: CLI stats vs statistics alias

- **Status:** Candidate
- **Severity:** Low
- **Location:** docs/user-guide/CLI.md:57, docs/reference/CLIReference.md:59
- **Authority:** apps/mboot-cli/CliParser.cpp — both "stats" and "statistics" map to CommandType::Statistics
- **Evidence:** Both forms work. Source help text only shows "stats". Completion scripts only list "stats". CLI.md uses "statistics". CLIReference.md documents both. Inconsistency across surfaces.
- **Recommendation:** Standardize on `stats` as primary form in docs, noting `statistics` as an accepted alias.
- **Target B3 Patch:** Patch 2
